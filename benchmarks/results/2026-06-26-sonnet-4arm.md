# Sonnet 4.6, four arms: baseline vs caveman vs ponytail vs capybaraa

Date: 2026-06-26. Model: `claude-sonnet-4-6`. A small, deliberately cheap run that puts
capybaraa next to its honest peers so the differences are visible, not asserted.

Arms, each loaded the way it actually ships:
- `baseline`: the bare agent, no plugin, no extra prompt. The reference everything is measured against.
- `caveman`: the caveman skill (prose-compression, MIT) as a system prompt. Control for "is the win just talking less?"
- `ponytail`: the ponytail plugin (4.7.0) loaded via `--plugin-dir`, its SessionStart hook live. Single-axis: minimal LOC.
- `capybaraa`: this plugin, activation verified per cell (`act='on'`).

Two batches, kept separate on purpose. The build axis (clear tasks, everyone builds) and the
clarify axis (ambiguous tasks, the point is whether you ask) must never be averaged together,
or an arm that asks and stops would look fake-lean on LOC. Build batch: `feat-rating`,
`feat-export`, 4 arms, n=2. Clarify batch: `clarify-settings`, `clarify-export`, 3 arms
(caveman dropped, it is a prose skill, not a clarify one), n=2.

## Numbers

### Clear build tasks (percent of the bare baseline, lower is leaner / cheaper / faster)

| vs baseline | LOC | output tokens | cost | wall time | complete |
|---|--:|--:|--:|--:|--:|
| caveman | 79% | 93% | 105% | 109% | 3/3 |
| **ponytail** | **47%** | 83% | 111% | 95% | 3/3 |
| **capybaraa** | 83% | 100% | 116% | 109% | 3/3 |

Baseline totals over the two tasks: 58 LOC, 2072 output tokens, $0.208, 40s. Per-task median LOC:

| task | baseline | caveman | ponytail | capybaraa |
|---|--:|--:|--:|--:|
| feat-rating (star widget) | 41 | 29.5 | 17 | 32.5 |
| feat-export (CSV download) | 17 | 16.5 | 10.5 | 15.5 |

Read it straight: **ponytail is the leanest on raw lines, by a lot** (that is its whole job).
capybaraa is modestly leaner than the bare agent (-17%) and caveman lands between. Every arm
shipped the feature fully (3/3 completeness judge), so none of the LOC cuts are stubs.

The honest cost story: on tasks this clear, **every assisted arm costs a little more than a
bare agent**, because they all inject a ruleset the baseline doesn't carry. capybaraa is the
priciest here (+16%): the badge, the sign-off, the done-gate are real output. We are not hiding
that. Note total token count is dominated by cached system-prompt injection, so it flatters no
one and we chart output tokens (what the model actually wrote) plus cost (the real bill) instead.

### Ambiguous tasks (CLARIFY judge 0-3, higher = asked the spec-deciding questions first)

| arm | median | premature LOC on "add settings persistence" |
|---|--:|--:|
| baseline | 0.0 | ~308 |
| ponytail | 0.0 | ~20 |
| **capybaraa** | **1.0** | **0** |

This is where capybaraa earns its cost. Handed "Add user settings persistence to this web app"
with no spec, the bare agent invented one and wrote ~300 lines of a settings manager; ponytail,
true to form, wrote a terse ~20-line guess; capybaraa wrote zero lines and asked which settings,
where to persist them, and per-device or synced first. On that ticket capybaraa cost less than
the baseline's wrong build ($0.17 vs $0.26): asking beats rebuilding.

## Honest read

- **No arm wins everything, and we don't claim capybaraa does.** On clear, reducible builds
  ponytail is the leanest and the bare agent is the cheapest. capybaraa's edge is the ambiguous
  case: it asks before it guesses, so it doesn't spend a session building the wrong thing.
- The LOC saving on clear builds is real but **modest** (~17% here) and not capybaraa's headline.
  "Smart tokens" means spending where it buys correctness (a question that prevents a 300-line
  rebuild), not minimizing lines for their own sake.
- capybaraa's CLARIFY median is 1.0, not a perfect 3.0: it asks, sometimes tersely, and one run
  referenced the questions without listing them. The win is over baseline and ponytail, which
  asked nothing (0.0).
- **Directional, not a leaderboard.** n=2, two build tasks, two clarify tasks, one model. CLARIFY
  and completeness are LLM judges (fixed model `claude-sonnet-4-6`, temp 0, published rubric, run
  with no plugins so no arm grades itself; every verdict lists asked vs missed).

Reproduce:

```
python3 run.py --selftest && python3 judge.py --selftest-offline
python3 run.py --task feat-rating,feat-export --arms baseline,caveman,ponytail,capybaraa --models sonnet --runs 2
python3 judge.py --complete-run runs/<build-stamp>
python3 run.py --task clarify-settings,clarify-export --arms baseline,ponytail,capybaraa --models sonnet --runs 2
python3 judge.py --run runs/<clarify-stamp>
cp runs/<clarify-stamp>/clarify.json runs/<build-stamp>/clarify.json
python3 chart.py runs/<build-stamp> ../../assets/benchmark.svg
```

Prereq for the peer arms: ponytail installed (or `PONYTAIL_PLUGIN_DIR` set); caveman ships
vendored at `agentic/caveman-SKILL.md`, no install needed.
