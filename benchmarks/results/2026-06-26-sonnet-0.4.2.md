# Sonnet 4.6, capybaraa 0.4.2, focused clean run

Date: 2026-06-26. Model: `claude-sonnet-4-6`. Arms: `baseline` (bare agent, no plugin) vs
`capybaraa` (on). A small, deliberately cheap run: 4 tasks (the two that actually
differentiate the plugin, clarify and leanness), n=3, single batch, 3 workers, no rate-limit
hits. Activation verified per cell. The broader 0.4.0 run (all tasks, gates) still stands for
the no-regression result; this run re-measures the headline behaviors cleanly.

## Numbers

CLARIFY (0-3, higher = asked more before coding), n=6 per arm:

| arm | median |
|-----|--------|
| baseline | 0.0 |
| capybaraa | 1.0 |

Leanness, median LOC on the feature tasks (lower is leaner), completeness 3/3 for both arms:

| task | baseline | capybaraa | delta |
|------|----------|-----------|-------|
| feat-rating (star widget) | 33 | 29 | -12% |
| feat-export (CSV) | 20 | 16 | -20% |

No regression: in the broader 0.4.0 run all nine deterministic gates (correct, safe, reuse,
native, optimal, economy, sync, hygiene, ran_check) were 100% for both arms; not re-run here.

Cost/time: on the ambiguous settings ticket capybaraa was cheaper and faster (asks instead of
building the wrong thing, $0.17 vs $0.22); on the build tasks it is a little pricier (the
clarify + done-gate overhead, e.g. feat-export $0.114 vs $0.099).

## Honest read

- The code saving is real but **modest and task-dependent**: ~12-20% on these two over-build
  features. An earlier broader run (2026-06-25) happened to catch -39% on the star widget;
  that is the high end of real run-to-run variance, not a typical figure. The defensible claim
  is "12-20% on real features, near zero on already-minimal code, fully complete either way."
- capybaraa asks the spec-deciding questions more often than the bare agent by default
  (1.0 vs 0.0), which is the behavior the pillar targets. It is not a perfect 3/3, it asks
  but tersely; the win is over a baseline that mostly does not ask at all outside plan mode.
- Directional, not a leaderboard: 4 tasks, one model, n=3. CLARIFY is an LLM judge (fixed
  model, published rubric, every verdict lists asked vs missed).

Reproduce: `python3 run.py --selftest`, then
`python3 run.py --task clarify-settings,clarify-export,feat-rating,feat-export --arms baseline,capybaraa --models sonnet --runs 3`,
then `judge.py --run` / `--complete-run`, then `chart.py runs/<stamp> ../../assets/benchmark.svg`.
